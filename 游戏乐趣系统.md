# RandomItemPVP 游戏乐趣系统

## 🎁 击杀奖励系统

### 基础奖励（每次击杀必得）

**立即回血：**
- 💚 回复 3 颗心（6 HP）
- ✅ 鼓励主动进攻，降低保守打法
- ⚔️ 支持连续作战能力

**随机物品奖励：**
- 📦 每次击杀获得 **3 个随机稀有物品**
- 🎲 物品池包括：
  - 金苹果
  - 末影珍珠
  - 箭矢
  - 光灵箭
  - 盾牌
  - 金胡萝卜
  - 经验瓶

**增益效果（30%概率）：**
- ⚡ **速度**（Speed I，10秒）
- 💪 **力量**（Strength I，10秒）
- 💖 **再生**（Regeneration I，10秒）
- 🛡️ **抗性**（Resistance I，10秒）

## 🎮 游戏平衡性

### 击杀奖励平衡

| 要素 | 设计目的 |
|------|----------|
| 回血 3 颗心 | 支持连续作战，但不会满血 |
| 3 个随机物品 | 适度奖励，不会过于强势 |
| 30% 增益概率 | 锦上添花，不是必得 |
| 10秒增益时长 | 短期优势，需要把握时机 |

### 连杀系统平衡

| 连杀数 | 获得难度 | 奖励价值 | 风险 |
|--------|----------|----------|------|
| 3 连杀 | 中等 | 中等（金苹果） | 成为焦点 |
| 5 连杀 | 较难 | 高（附魔金苹果） | 众矢之的 |
| 7 连杀 | 困难 | 极高（不死图腾） | 全服狙击 |
| 10 连杀 | 极难 | 顶级（下界合金） | 终极目标 |

**平衡点：**
- 高连杀意味着高收益，但也成为全场目标
- 终结高连杀玩家也能获得荣誉
- 鼓励挑战强者，避免单方面碾压

### 空投系统平衡

**时间平衡：**
- 60 秒间隔给所有玩家机会
- 30 秒首次延迟确保游戏稳定开局

**位置平衡：**
- 随机位置避免固定点守株待兔
- 信标标记确保可见性

**内容平衡：**
- 稀有但不绝版
- 保底机制确保价值
- 不会直接决定胜负

## 🔧 技术实现

### 击杀奖励
```java
- 监听 PlayerDeathEvent
- 检测 killer 不为 null
- 回血：setHealth(current + 6.0)
- 物品：从奖励池随机选择
- 增益：30% 概率添加药水效果
```

### 连杀系统
```java
- HashMap<UUID, Integer> 追踪连杀数
- 每次击杀 +1
- 达到阈值触发特殊奖励
- 死亡时清除记录
- 全服播报 + 标题显示
```

### 空投系统
```java
- ScheduledTask 定时任务
- 60秒间隔，30秒首次延迟
- 随机位置生成箱子
- 信标光柱标记
- 监听箱子打开事件
```

## 📝 配置信息

这些系统的参数已内置在代码中，暂不支持配置文件修改。

如需调整参数，请修改对应管理器中的常量：

**RewardManager.java：**
```java
KILL_HEAL_AMOUNT = 6.0;       // 击杀回血量
KILL_REWARD_ITEMS = 3;        // 击杀奖励物品数量
```

**AirdropManager.java：**
```java
AIRDROP_INTERVAL_TICKS = 1200L;    // 空投间隔（60秒）
FIRST_AIRDROP_DELAY = 600L;        // 首次延迟（30秒）
```
